# Plan 11-01: Create Vertical Right Sidebar with View Tabs

## Overview

Restructure App.tsx layout to move view tabs (Kanban, DAG, Context) from horizontal header tabs to a vertical right sidebar. The main content area will expand to fill the available width, with the sidebar providing view switching.

## Why

Current layout has horizontal tabs at top which wastes vertical space. A vertical right sidebar follows modern IDE patterns and frees up the header for other controls. This is the foundation for the new layout structure.

## Current State

- App.tsx has header with horizontal tabs + auth + new feature button
- Main content area shows one view at a time
- View switching via useViewStore requestViewChange()
- Tabs configuration: Kanban, DAG, Context

## Target State

- Vertical right sidebar (w-12 collapsed, icons only)
- Main content area fills remaining width
- Sidebar icons for Kanban/DAG/Context with active indicator
- Header simplified (removed tabs, keeps auth + new feature for now)
