# Plan 11-02: Create Bottom Status Bar Component

## Overview

Create a bottom status bar component to display application state information. The status bar will be a fixed-height bar at the bottom of the app, providing space for auth indicator, git status, and other runtime information.

## Why

Status bars are standard UI patterns for showing persistent application state. Moving auth and git info to the status bar keeps the header clean and provides a consistent location for system status.

## Current State

- No status bar exists
- Auth indicator in header (to be moved in 11-03)
- No git status display

## Target State

- Fixed bottom status bar (h-8) with bg-gray-800
- Left section: Project info / git branch
- Right section: Auth status (placeholder for 11-03)
- Border-top separator from main content

   - src/renderer/src/components/Layout/StatusBar.tsx

   - Flex layout with left/right sections
   - Left: placeholder text "DAGent" for now
